## 🚀 快速登基（Getting Started）

**环境准备：** 确保你的 Windows 系统安装了 [**.NET 8.0 运行环境**](https://dotnet.microsoft.com/download/dotnet/8.0/runtime?utm_source=getdotnet&utm_medium=referral)。

**微服私访：** 将你的资源路径填入 `config/paths.txt`（或者直接在 UI 界面点击设置）。

**登基大典：** 双击 `CyberConcubineSelection.exe`。

**开始选妃：** 点击那个最显眼的“抽选”按钮，迎接今晚的惊喜。

## 🎯 拒绝“降智”随机：全链路甄选引擎

别把这和播放器的“随机播放”混为一谈。我们的算法有 6 层“安检”：

**多路径管理：** 支持 6 个硬盘槽位并行加载。

**血统净化：** 黑白名单机制。自动隔离不想要的文件夹；一键狙击 AI超分 (Iris/Prob) 或 无码 (Uncensored) 核心资源。

**动态加权：** 支持“喜新厌旧”或“陈年佳酿”模式。给心爱的资源点个“心”，抽中概率原地起飞。

**绝对避嫌：** 刚看过的，N 天内自动屏蔽，彻底告别审美疲劳。

### 🖼️ 图片功能：图册也有规矩

别以为这套系统只会挑视频，不管你是漫画还是图包，图片资源同样纳入完整调度。

**图片抽选：** 从启用路径中统一抽选，不是随手乱翻相册。  
**黑名单过滤：** 支持按目录名、文件名和自定义关键词排除不想看到的内容。  
**历史记录：** 每次查看都会留档，可读取最近图片，也可按时间范围回顾。  
**最近避嫌：** 支持按最近天数或最近条数排除，避免刚看过又重复翻牌。  
**偏好联动：** 支持“仅收藏”“从未看过”等过滤条件，图片和视频一套规矩。  
**时间权重：** 支持“喜新厌旧”或“陈年佳图”模式，让抽图也有偏好，不靠纯随机。

这是我们最硬的底线。

### 🎨 个性化定制：背景和按钮音效都能自己配

程序支持通过本地资源自定义部分界面的视觉和交互反馈。

**窗口背景：**  
主界面以及部分功能窗口会从 `img/` 随机读取背景资源。  
目前代码里实际用到的包括：

- `img/Main`：主界面背景
- `img/Config`：配置界面背景
- `img/History`：历史记录窗口背景
- `img/Report`：年度报告及部分统计窗口背景

支持的背景格式为：

- 静态图片：`.jpg` / `.jpeg` / `.png`
- 视频背景：`.mp4`

其中部分统计窗口使用的是静态背景版本，不会读取视频。

**按钮音效：**  
主界面 5 个核心按钮支持自定义点击音效，会从 `audio/` 文件夹中随机匹配本地音频文件：

- `btn1.wav` / `btn1.mp3`：配置按钮
- `btn2.wav` / `btn2.mp3`：随机翻牌按钮
- `btn3.wav` / `btn3.mp3`：随机图片按钮
- `btn4.wav` / `btn4.mp3`：历史记录按钮
- `btn5.wav` / `btn5.mp3`：年度报告按钮

如果同一编号下存在多个匹配文件，程序会随机播放其中一个；如果没有找到对应音效，则该按钮保持静默。并且适配多种命名方式。

### 📁 config 文件夹说明：圣旨、起居注与内务府档案

`config/` 是整个系统的本地档案库。  
模式、路径、过滤规则、收藏、历史记录、统计缓存，基本都在这里落盘。没有云端，没有同步服务器，你的全部“后宫内政”都在这个目录里。

以下是程序实际会读取或生成的配置文件：

- `paths.txt`  
  路径配置文件。记录多个资源槽位的目录路径以及启用状态，是整个资源调度系统的入口。

- `mode.txt`  
  当前模式配置。记录主模式和当前激活的过滤条件，比如图片模式 / 视频模式、仅收藏、从未看过等。

- `exclusion.txt`  
  最近排除规则配置。控制“最近 N 天”或“最近 N 条”避嫌逻辑，以及文件夹模式下的最小数量阈值。

- `custom_keywords.txt`  
  自定义筛选关键词配置，让过滤逻辑更贴近个人口味。

- `black_list.txt`  
  黑名单配置。用于屏蔽你不想再看到的目录、编号、标签或关键词，是“血统净化”的第一道关卡。

- `stop_words.txt`  
  停用词配置。仅优化文件夹名字显示，减少无意义词汇干扰，提高过滤判断的准确度。

- `favorites.txt`  
  收藏列表。保存你手动点过“心”的资源路径，供“仅收藏”模式和加权逻辑使用。

- `History.log`  
  核心历史记录文件。图片和视频的记录都会写进这里，后面的统计、热力图、年度报告基本都靠它起家。

- `BackgroundUsage.log`  
  背景使用记录。用于统计背景资源的使用情况，供背景热力图和相关报告使用。

- `path_usage.log`  
  路径使用记录。用于统计各资源路径的活跃度，后续可以生成路径热力图或路径偏好分析。

- `super_resolution_keywords.txt`  
  AI 超分关键词配置。用于识别和过滤 `Iris`、`Prob` 一类超分资源。

- `uncensored_keywords.txt`  
  无码关键词配置。用于识别和筛选 `Uncensored` 等特殊资源标签。

- `teacher_collection_cache.json`  
  烧鸡收藏分布缓存。用于保存相关统计结果，减少重复扫描和重复计算。**仅在用户主动点击后获取扫描结果**

- `video_quality_snapshot.json`  
  视频质量快照缓存。用于记录视频清晰度或质量分布，方便统计窗口直接读取。**仅在用户主动点击后获取扫描结果**

- `video_codec_cache.json`  
  视频编码缓存。用于保存ffprobe视频编码扫描结果，避免每次都重新分析。**仅在用户主动点击后获取扫描结果**

- `unwatched_video_cache.json`  
  未观看视频缓存。用于加快“从未看过”相关统计或排行窗口的加载速度。

- `video_inventory_snapshots_年份.json`  
  年度资源清单快照。按年份保存视频库存快照，用于年度报告和库存变化统计。

- `favorite_year_stats.txt`  
  年度收藏统计。保存本年度收藏相关的统计结果。

- `favorite_image_pack_year_stats.txt`  
  年度图片包收藏统计。用于记录图片收藏包在年度维度下的使用与偏好情况。

# 白名单与噪点识别交互说明

## 优先级

1. **白名单检查（最高优先级）**
2. 噪点识别
3. 父文件夹白名单检查

## 处理流程

### 基本流程
```
1. 从文件的直接父文件夹开始
2. 检查是否在白名单 → 是 → 停止，显示该名字
3. 检查是否是噪点 → 是 → 向上一级，重复步骤2
4. 不是噪点 → 检查父文件夹是否在白名单
   - 父文件夹在白名单 → 向上一级，显示父文件夹名
   - 父文件夹不在白名单 → 停止，显示当前名字
```

## 实际案例

### 案例1: 三层结构 - 系列/女优/编号

路径：`ssni/三上悠亚/ssni-618/ssni-618.mp4`

#### 情况A: SSNI 不在白名单
```
步骤1: ssni-618
- 白名单检查: ❌
- 噪点检查: ✅ (匹配编号格式)
- 向上到: 三上悠亚

步骤2: 三上悠亚
- 白名单检查: ❌
- 噪点检查: ❌
- 父文件夹检查: ssni 不在白名单 ❌
- 停止

结果: 显示 "三上悠亚" ✅
```

#### 情况B: SSNI 在白名单
```
步骤1: ssni-618
- 白名单检查: ❌
- 噪点检查: ✅ (匹配编号格式)
- 向上到: 三上悠亚

步骤2: 三上悠亚
- 白名单检查: ❌
- 噪点检查: ❌
- 父文件夹检查: ssni 在白名单 ✅

步骤3: ssni
- 白名单检查: ✅
```

## 白名单的作用

### 1. 保护重要文件夹名
```
如果某个文件夹名被误判为噪点，可以加入白名单保护
```

### 2. 控制显示层级
```
- 想显示女优名 → 不要把系列名加入白名单
- 想显示系列名 → 把系列名加入白名单
```

### 3. 处理特殊格式
```
如果某个编号格式没被正确识别，可以手动加入白名单
```

## 常见白名单配置

### 配置1: 显示女优名（推荐）
```
白名单: 空 或 只包含特殊文件夹名

路径: ssni/三上悠亚/ssni-618/video.mp4
显示: 三上悠亚

路径: FC2/橋本ありな/FC2-PPV-1234567/video.mp4
显示: 橋本ありな
```

### 配置2: 显示系列名
```
白名单: FC2, SSNI, SNIS, IPZ, MIDD, ABP, 等

路径: ssni/三上悠亚/ssni-618/video.mp4
显示: ssni

路径: FC2/橋本ありな/FC2-PPV-1234567/video.mp4
显示: FC2
```

### 配置3: 混合配置
```
白名单: FC2, ERIKA (只保护特定系列)

路径: ssni/三上悠亚/ssni-618/video.mp4
显示: 三上悠亚 (SSNI 不在白名单)

路径: FC2/橋本ありな/FC2-PPV-1234567/video.mp4
显示: FC2 (FC2 在白名单)

路径: erika/ERIKA-001/video.mp4
显示: erika (ERIKA 在白名单)
```

## 父文件夹白名单检查的意义

这个设计允许用户灵活控制显示层级：

```
路径: ssni/三上悠亚/ssni-618/video.mp4

如果用户想看到女优名:
- 不要把 SSNI 加入白名单
- 结果: 显示 "三上悠亚"

如果用户想看到系列名:
- 把 SSNI 加入白名单
- 结果: 显示 "ssni"
```

这样用户可以根据自己的组织习惯来配置白名单，非常灵活！

## 总结

1. 白名单拥有最高优先级
2. 噪点识别会自动向上查找
3. 父文件夹白名单检查提供了额外的灵活性
4. 用户可以通过配置白名单来控制显示层级
5. 推荐配置：不加入系列名，显示女优名（更有意义）
